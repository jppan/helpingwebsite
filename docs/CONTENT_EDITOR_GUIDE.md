# Content Editor Guide

## What You Can Edit Safely

- Guide content in `content/guides/**`
- Emergency items in `content/emergency.ts`
- Common fixes in `content/common-fixes.ts`
- Rico profile copy in `content/profile.ts`
- Homepage/docs hub curation in `data/featured-guides.ts`

## What To Avoid

Do not edit these unless you are intentionally changing the app structure:

- `src/app/**`
- `src/components/ui/**`
- `src/lib/**`
- `src/app/globals.css`
- `next.config.ts`
- `.github/workflows/**`

## Writing Rules

- Use plain language
- Prefer problem titles over technical labels
- Keep paragraphs short
- Use numbered steps
- Keep warnings calm and direct
- Keep fallback guidance explicit

## Placeholder Rule

Any missing screenshot or illustration must stay a bright red placeholder until a real asset is ready.
